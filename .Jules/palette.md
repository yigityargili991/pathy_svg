## 2024-05-18 - Improve tooltip accessibility
**Learning:** The CSS-based tooltips in pathy-svg rely entirely on the ':hover' pseudo-class. This means they are completely inaccessible to users navigating via keyboard (using Tab to focus elements) or screen readers (which often trigger focus rather than hover). This is a common accessibility issue with CSS-only interactive elements.
**Action:** Update the CSS injected for tooltips to use both ':hover' and ':focus' (or ':focus-visible') selectors. Ensure the SVG elements receiving tooltips can actually receive focus by adding a 'tabindex' attribute (e.g., 'tabindex="0"'). Also, adding an 'aria-label' matching the tooltip text is good practice for screen reader users, so they don't even need to trigger the tooltip visually to get the information.
