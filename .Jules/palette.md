## 2024-05-18 - CSS Tooltips Accessibility Pattern
**Learning:** CSS-only hover tooltips in SVGs created a significant accessibility gap because keyboard users couldn't trigger them. Native SVG `<title>` works for screen readers, but CSS custom tooltips (`[data-tooltip]:hover + .tooltip`) hide content from keyboard navigation. Adding `tabindex="0"`, `aria-label`, and extending the CSS selector to `[data-tooltip]:focus + .tooltip` creates a reusable pattern that fixes this without JS. Adding `:focus-visible` ensures clear visual feedback only for keyboard users, preventing unwanted focus rings on mouse click.
**Action:** When using CSS sibling selectors for interactive SVG states (like tooltips or highlights), always pair `:hover` with `:focus`, add `tabindex="0"`, and set the appropriate ARIA properties (`aria-label` or `aria-describedby`) directly on the interactive element.
