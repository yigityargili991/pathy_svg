## 2024-05-24 - [CRITICAL] Prevent SSRF Vulnerability and Redirect Bypass in Remote SVG Fetching
**Vulnerability:** Server-Side Request Forgery (SSRF) allowed `SVGDocument.from_url` to fetch and parse internal/private network assets. An attacker could supply a URL pointing to local/private IP addresses or one pointing to an external server they control that returns a 302 Redirect to a private IP, bypassing naive initial host checks.
**Learning:** Python's `urllib.request` performs no hostname/IP validation against SSRF and seamlessly follows redirects by default. To safely fetch user-supplied URLs, one must: 1) resolve the hostname to IP addresses via `socket` and filter out restricted blocks using `ipaddress`; 2) install a custom `HTTPRedirectHandler` that disables redirects completely to prevent redirect-based SSRF bypass.
**Prevention:** Always explicitly validate resolved IP addresses. Never trust `urllib.request.urlopen` default settings. Always use a custom opener with redirects disabled (`urllib.request.build_opener(NoRedirectHandler())`) when downloading untrusted assets.
